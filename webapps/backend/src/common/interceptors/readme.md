
# Interceptor (ในบริบทของ NestJS หรือ Angular) คือ "ตัวคั่นกลาง" ที่คอยดักจับและจัดการกระบวนการระหว่างที่ Request กำลังวิ่งเข้ามาหา Controller หรือตอนที่ Response กำลังจะถูกส่งกลับไปหา Client ครับ

เปรียบเทียบง่ายๆ เหมือนกับ "รปภ. ที่ยืนอยู่หน้าห้องทำงาน"
    ขาไป: รปภ. จะตรวจสอบเอกสาร หรือจดบันทึกว่าใครมาหา (Logging)
    ขากลับ: เมื่อคนในห้องทำงานเสร็จ รปภ. อาจจะช่วยห่อของขวัญให้ก่อนส่งคืน (Transform Response) หรือจดเวลาว่าคนนั้นใช้เวลาคุยนานแค่ไหน (Performance Monitoring)

หน้าที่หลักของ Interceptor (The Power of Interceptors)
    Interceptor ได้รับแรงบันดาลใจมาจากแนวคิด Aspect-Oriented Programming (AOP) ซึ่งช่วยให้เราจัดการงานที่ซ้ำซ้อนได้ในที่เดียว:
    Bind extra logic: เพิ่มคำสั่งพิเศษก่อนหรือหลังการทำงานของฟังก์ชัน (เช่น การเริ่มเก็บ Log และจบการเก็บ Log)
    Transform the result: เปลี่ยนรูปแบบข้อมูลที่ส่งกลับมา (เช่น การครอบข้อมูลด้วยก้อน JSON { data: ... })
    Transform the exception: เปลี่ยนรูปแบบ Error ที่เกิดขึ้นให้เป็นแบบที่เราต้องการ
    Extend basic function behavior: เพิ่มความสามารถให้ฟังก์ชัน เช่น การทำ Caching (ถ้ามีข้อมูลในเครื่องอยู่แล้ว ก็ให้ส่งกลับไปเลย ไม่ต้องไปรัน Logic ใน Controller)
    Override a function: ตัดตอนการทำงานบางอย่างตามเงื่อนไขที่กำหนด

โครงสร้างและวิธีการทำงาน
    Interceptor จะมี Method สำคัญที่ชื่อว่า intercept() ซึ่งรับพารามิเตอร์ 2 ตัว:
        ExecutionContext: ข้อมูลเกี่ยวกับ Request นั้นๆ (เช่น มาจาก Route ไหน, ใครเป็นคนส่งมา)
        CallHandler: เป็นตัวควบคุมว่า "จะให้โปรแกรมทำงานต่อไปยัง Controller หรือยัง?" โดยใช้คำสั่ง .handle()
